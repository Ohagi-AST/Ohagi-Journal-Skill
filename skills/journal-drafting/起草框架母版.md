# 英文期刊起草框架（通用母版 / Journal-Level Drafting Framework）

> **这是什么**：一份「研究已完成 → 开始正式英文期刊级写作」这一步的可复用规则书。
> **怎么用**：每开一篇新论文，**只重填「第 0 层 SPEC 槽位」**；「通用硬规则（1–9）」永不改、所有论文共用。
> **不要做的事**：第 0 层没填满之前，不许动笔写正文。这是硬闸。
>
> 设计来源：把 agent harness 的 *Rules-as-spec（1–7）+ Verification-loop（8）+ Memory（9）* 三件套落到论文成稿。
> 维护原则：通用规则区尽量稳定；每篇的具体内容只进第 0 层和「本篇决策日志」，不要污染通用区。

---

## 第 0 层｜每篇填空的 SPEC 槽位（Phase 0，不填满不动笔）

> 每开一篇新论文，复制下面这块到该论文文件夹里，逐项填实。**带 ⛔ 的项空着 = 禁止进入正文写作。**

```
论文代号：________
一句话研究问题（≤1句，钉死，全程不许漂）：________

⛔ 目标期刊（具体到刊名，不许写"顶刊/某领域好刊"这种抽象目标）：________
⛔ 目标层级现实性判断（这篇的贡献/数据/识别，诚实对得上这个刊吗？对不上就降档）：________
⛔ 结构母本 3–5 篇（"识别结构/主题最像"的已发表论文，当骨架去填，不从零发明）：
    1. ________
    2. ________
    3. ________
字数 / 格式 / 章节数上限：________
写作语言（英文 / 日文-投稿 / 日文-修論；选日文则末段启用「日语优化步骤」+净化层，见编排器 SKILL 第 3.5 步）：________
引用样式（如 AEA author-date）：________

⛔ 一句话 contribution（"本文做了 X，区别于最近的 ____，多出了 ____"）：________
本篇专属红线（本篇识别强度决定的禁区，写成可执行的一句话）：________
    例：识别为内生选择/关联 → 全文禁止 cause/effect/impact 等因果措辞，改用 associated with / consistent with
数据出处一句话（可复现的来源）：________
```

**接缝·结构母本可升级为「动态写作层」**（用 `journal-adapt` skill）：
- **是否调用的判断**（泛用，对任何论文一致）：同时满足 ↓ 才值得调——① 有一个**明确的目标期刊**；② 能拿到该刊 **5–8 篇真论文**当语料；③ 是**英文**写作。
- 满足 → 把"挑母本"升级成"萃取该刊写作文化"，产出 `dynamic_writing_skill.md`，逐节写时它当**动态层**、本母版当**静态层（P4）**叠加使用。
- 不满足（无明确目标刊 / 凑不齐语料 / 非英文，如学位论文按格式规程写）→ **跳过**，只用本母版静态层即可。

## 适用档位：期刊投稿 vs 学位论文

> 本框架默认按"期刊投稿"全配置。**若最终目标是学位论文（修士/博士论文）**，按下表**跳过/软化期刊专属环节**——通用骨架全部保留。编排器 `journal-drafting` 会在开头**隐形判断档位**（不确定先问用户）。

| 环节 | 期刊投稿档 | 学位论文档 |
|---|---|---|
| 第 0 层「目标」 | 目标刊 + 层级现实性 + 结构母本 | 学校格式规程 + 审查/答辩委员；**免**层级现实性判断 |
| 动态写作层（journal-adapt） | 视情况调用 | ⏭️ 跳过（无期刊写作文化可学） |
| 文献宽度扩展 | 视情况 | 可选（先行研究宽度不足时才用，不强求穷尽） |
| 新规性（规则 3） | 1–3 条、扛点名最近邻、禁裸 first-to | **软化**：清晰研究问题 + 适度贡献即可 |
| 系数→经济量级（规则 6） | 建议 | 可选 |
| **图表蓝图（规则 6 扩展）** | 跟目标刊全配置（house-style/叙事弧/头牌） | **核心照走**（蓝图·三线表·视觉自检·不造数·write-to-exhibit）；house-style 与引出语跟**学校規程·目标语言**、渲染工具可选。详见 `references/图表设计与叙事.md`「适用档位」 |
| 引用穷尽核验（reference-checker） | 投稿前穷尽 + 分级 | **轻量**：不编造 + 标【待核】+ 抽查关键引用 |
| 审稿人预演（规则 8） | 顶刊 referee 视角 | **降级**为答辩委员视角，或跳过 |
| 期刊专属：匿名化 / 投稿体例 / 字数硬限 / 复现包 | 适用 | ⏭️ 跳过，改用学校规程 |
| **通用骨架：规则 1 骨架 · 2 文献定位 · 4 主张≤识别 · 5 不编造 · 7 风格与语气纯净 · 8 单节自查 · 9 防漂移** | ✅ | ✅ **全部保留** |

> 一句话：**学位论文＝通用骨架照走，期刊级"打磨/守门"环节按需跳过。**

# 通用硬规则（以下 1–9 所有论文共用，永不改）

## 1. 叙事框架（Narrative & Structure）

- **一篇 = 一个核心问题。** Intro 前两三段必须让读者说得出"这篇要回答什么"。回答不出 = 框架坏了，停笔重搭。
- **经济学实证论文的标准骨架**（按目标刊母本微调，但别缺环）：
  `Intro → Positioning（文献定位）→ Data/Institutional background → Identification/Empirical strategy → Results → Robustness → Mechanism → Conclusion`
- **Intro 的固定配方**（顶刊 intro 不是创作，是填配方）：
  `Hook（为什么重要）→ Puzzle/Gap（已知什么、缺什么）→ This paper（本文做了什么）→ How（数据+识别一句话）→ Findings（主结果）→ Contribution（1–3条）→ Roadmap`
- 每节开头一句"本节做什么、得到什么"，不让读者自己拼。

**各节配方（缺的标【待核】/【MISSING】，别脑补）：**
- **Data 节**：来源与制度背景 / 样本期与观测单位 / 样本筛选与最终样本量 / 被解释·处理·控制·固定效应 / 变量单位与变换 / 对解读重要的描述统计。
- **識別戦略 节**：估计方程 / 识别变异 / 对照逻辑 / 固定效应与控制 / 标准误聚类层级 / 关键识别假设 / 威胁与对应检验。
  ⛔ **不要把固定效应本身当识别——必须说明固定效应之后还剩什么变异在干活**（重力面板尤其要命）。
- **Results 节顺序**：方向与量级 → 统计显著性 → 经济显著性 → 跨设定稳健性 → 与假设的关系 → 必要 caveat。⛔ 没有 baseline mean / 标准差就别编经济显著性。呈现顺序应与**图表蓝图的叙事弧**一致（规则 6 / `references/图表设计与叙事.md`）。
- **Robustness / Heterogeneity / Mechanism**：稳健性必须回应某个**具体威胁**；异质性必须有**经济理由**；机制只能写"渠道证据"，不是完整因果链的证明。
- **分领域约定**：不同子领域呈现习惯不同（宏观/金融/贸易引力/理论…）→ 按需读 `references/领域写作约定.md`，别把通用骨架硬套。

## 2. 文献综述（Literature Review = 定位，不是罗列）

- 综述的功能是**给本文定位**，不是给读者做读书报告。**禁止 annotated-bibliography 式逐篇复述。**
- **每一簇文献必须收束到一句"缺了什么 → 那就是我的 gap"。** 没有这句收束的段落删掉。
- 必引领域内**绕不开的前作**（最接近你的那几篇）。漏引最近的近邻 = referee 一票否决高发区。
- 综述里出现的每一篇，都要能回答："它和本文什么关系？支持/对照/被超越？"
- **产出一张「文献地图」表**：簇 | 关键引用 | 该簇已确立的发现 | 本文相对它的贡献。先有表、再写散文。（同一"先有表、再写散文"原则，经规则 6 推广到全文结果图表——见 `references/图表设计与叙事.md`。）
- 句式：✅"This paper contributes to three strands of literature." / "Relative to this work, we…" / "The closest papers are …, which…"；❌"fills an important gap"（不说哪个 gap）/ ❌空洞开头"It is important to note that…"。
- **宽度不足时**（锚点够、但综述宽度不够）→ 用 `references/文献宽度扩展.md` 从锚点做引用图谱滚雪球（Semantic Scholar API），扩出按簇排序的候选，再写文献地图。

## 3. 新规性 / 贡献（Novelty & Contribution）

- 贡献写成 **1–3 条显式 bullet**，不藏在散文里。
- **每条贡献都必须对应一次真实的文献检索结果**，不是断言。
- ⛔ **禁止未经核实的 "we are the first to / 既存研究にはない / unlike all prior work"。** 要这么写，先有检索证据；没有就降级为"to our knowledge / 据我们所知"并准备好被问。
- 贡献必须能**扛住点名的最近邻**："相对 [作者 年份]，本文多出来的是 ____。"说不出这句 = 贡献不成立。

## 4. 主张强度纪律（Claims Discipline）★最关键、最通用

> **铁律：结论的语气 ≤ 识别的强度。** 这是本框架最硬的一条，适用任何论文。

- 设计是**关联 / 内生选择 / 描述性**时，**全文禁止因果措辞**（cause, effect, impact, leads to, drives, 因…导致…）。改用 `associated with / consistent with / predicts / 相関 / 整合的`。
- 因果语言只有在**识别策略真的支撑**时才解锁（且要点名是哪个变异在干活）。
- 写完每一节，自查一遍："这句话的强度，我的识别扛得住吗？"扛不住就降级措辞，**不是降级就是诚实，是保命**——审稿人最爱抓的就是 over-claim。
- 本规则的"本篇具体禁区"在第 0 层「专属红线」里实例化。

**识别审计清单**（写识别章、或自检因果措辞时逐条过；这是把"强度≤识别"从口号变操作）：
1. 控制和固定效应之后，是**什么变异**在识别这个参数？
2. 对照组是谁？
3. 需要哪些假设？
4. 这些假设是被**当作假设写**、还是被当成事实偷渡了？
5. timing / anticipation / 选择 / 溢出 处理了吗？
6. 标准误聚类在**处理变异的层级**上了吗？
7. 稳健性检验对得上**真实威胁**吗（还是答非所问）？
8. 文本有没有夸大因果解读？

**设计专用补检**：
- 面板 FE：固定效应后剩余的组内变异说清了吗？有没有 bad controls？序列相关？
- DID/事件研究：平行趋势、无预期、错期采纳（staggered）。
- IV：相关性、排他性、单调性、第一阶段、LATE 解读。
- RDD：连续性、操纵、带宽、协变量平衡、局部解读。

**因果措辞替换库**：设计不支持因果时 ⛔ 禁 `proves / demonstrates / X causes Y / 导致 / もたらす`，改用 → `The evidence is consistent with / The estimates suggest / The results support the interpretation that / This pattern points to / …と整合的`（完整句库见 `references/经济学写作风格.md`）。

## 5. 引用诚信（Citation Integrity）

- ⛔ **绝不编造**作者、年份、标题、期刊、页码、DOI。一个都不行。
- 拿不准的引用，**先写 `【待核】` 占位**，绝不让任何人（包括 AI）"补全"一个看起来合理的作者年份。
- 引用样式**全文一致**（按第 0 层指定的 style）。
- 每一条引用在定稿前都要能指向**真实可查的出处**；指不出 = 删除或核实，二选一。
- **双时点核验**（用 `reference-checker` skill）：
  - **靠前·地基引用**：第 0 层选定的结构母本 + 文献定位章必引的关键前作，**文献章动笔前先核一遍**——地基引用记错/造假会毁掉整个定位与新规性。
  - **靠后·穷尽终检**：投稿前对**整份**参考文献逐条核（英文走 DOI/Crossref/PubMed，中文走 CNKI/万方/官方页），**清空所有【待核】**，按 Verified/Minor/Major/Critical 分级。这是规则 5 的最终落地。
  - ⚠️ CNKI/万方常需登录、日文库（CiNii/J-STAGE）未覆盖 → 这部分会落到「Manual check」，需手动补。

## 6. 数据与可复现（Data & Reproducibility）

- 数据**出处写明、可追溯**（哪个库、哪个版本、什么时间口径）。
- **图表自含**：脱离正文也能看懂（标题、单位、样本、注释齐全）。
- 复现心态：变量定义、样本筛选、口径处理在文中或附录交代清楚，别让别人猜。

**图表先行 / 图表蓝图（全文级 → 见 `references/图表设计与叙事.md`）**：
- **先设计整套图表、再写散文**——段落服务图表、图表服务叙事；动笔正文前先填「图表蓝图」（exhibit 清单 + 叙事弧 + 锚定节 + 主/附放置），经用户确认再逐节写。反"文字提到就地塞张表"。
- **头牌按方法族**：缩减式/gravity→基准系数表（走列）；SCM/事件研究→主图；理论→核心命题。接 journal-adapt 则以目标刊治理族领头方式为准。
- **全文一套 house-style**：⛔ 表一律**三线表**（booktabs；**禁竖线/全框线/九宫格**）；星号/SE/小数/表注/题注位置全文一致；图灰度极简、题注在图下。渲染用 `scripts/`（esttab→booktabs / coefplot）。
- **视觉自检**：图表渲染成图、**亲眼看**一遍（三线表？无九宫格？对齐/表注达标？）才算定稿（并入规则 8）。
- **渲染≠造数**：渲染工具只格式化真实 `eststo` 估计，⛔ 绝不手写系数/SE/N/p（见下条与规则 5）。

**把回归表写成正文时的纪律**（优先准确，不优先漂亮）：
- **每句话绑定到一个表格列/面板/行**，不让读者猜你在说哪一格。
- ⛔ 绝不编系数、标准误、样本量、p 值、baseline mean、设定。
- 区分三件事：系数大小 / 统计显著 / 经济显著——别混。
- ⛔ 没有交互项或正式检验，**不得声称组间差异显著**（写"对…更大"，不写"对…显著更大"）。
- ⛔ 不把不显著/不稳定的结果说成 robust。
- **不显著也照实报**：零结果本身就是结果，照实写、别藏、别美化成"接近显著"。

**系数 → 经济量级**（重要结果不能只报"显著"，至少给一个基准，挑最贴该变量与识别来源的那一个）：
- **自然单位 / 真实政策幅度**（优先）：「X 上升 10% → Y 下降 3 个百分点」。
- **占被解释变量均值的比例** / **折算成标准差**（如"约 0.2 个标准差"）/ **对照已知干预或政策阈值**。
- **弹性**说明在均值、中位数还是弧弹性；**交互项/非线性/概率模型**先在有意义取值处算净效应/边际效应再写散文。
- ⛔ 没有均值/标准差/分位/政策基准就**别编**——从数据算、问用户、或标【待核】；也别机械堆砌一长串描述统计。

## 7. 文体 / 语域（Prose Register）

- 期刊英文语域：**简洁、主动语态优先、发现用现在时**（"We find that…"），避免口语和冗词。
- **hedging 纪律**：该 hedge 的（主张强度问题）一定 hedge，不该 hedge 的别滥用模糊词稀释结论。
- ⛔ **内部暗语 → 学术表述**：任何只在你备忘录/内部讨论里出现、没在正式计量文献里用过的词，**禁止进正文**。需要时为本篇建一张"内部词 → 英文学术表述"对照表（沿用你已有的用词规范习惯）。
- **共享写作风格层 → 见 `references/经济学写作风格.md`**：信息密度、禁套话、实质性过渡、段首定角色、用词 avoid→prefer 表、因果谨慎句库、数字/单位/图表注释。写任何一节正文前都套这一份。
- **稿件语气与正文纯净度 → 见 `references/稿件语气与正文纯净度.md`**：三通道分离（正文 / 作者备忘 / 任务日志，只有正文进稿子）+ 内部标签替换 + 段级 linter + 指针规则 + 硬拒模式。它是本条"内部暗语→学术表述"的系统化、可执行版，逐段跑（并入规则 8《单节自查》）。
- **日语轨专用（写作语言=日文才加载）**：① 净化层 `references/中文残留チェック.md`（中文打底残留三关：字形脚本 / 词汇同形异义 / 直译腔与标点）；② 文体层 `references/日本語学術文体.md`（投稿轨 + 修論轨两套日语学术文体）。由编排器 SKILL 第 3.5 步「日语优化步骤」驱动；非日语轨不加载。

## 8. 验证闸 / Checkpoint（Verification Loop，别 hope-driven）

> 把"检查"固化成可复现清单，而不是每次临场拍脑袋。

**每写完一节，过这张《单节自查》：**
- [ ] 这节开头说清"做什么、得什么"了吗？
- [ ] 主张强度 ≤ 识别强度？（规则 4）有没有手滑写成因果？
- [ ] 本节出现的引用，全部真实可查 / 已标【待核】？（规则 5）
- [ ] 字数 / 格式符合第 0 层 spec？
- [ ] 有没有内部暗语漏进去？（规则 7）
- [ ] 本节若产出/更新了图表：过了**视觉自检**吗（渲染成图→亲眼看：三线表/无九宫格/对齐/表注/与图表蓝图一致；规则 6 + `references/图表设计与叙事.md`）？

**投稿前过这张《总闸》：**
- [ ] 一句话研究问题 vs 全文——还是同一个问题吗？（防漂移）
- [ ] 1–3 条贡献，每条都有检索证据、都能扛住点名最近邻？（规则 3）
- [ ] 全文无编造引用，无残留【待核】？（规则 5）
- [ ] 全文主张强度自洽，无 over-claim？（规则 4）
- [ ] 综述每簇都收束到 gap，无逐篇罗列？（规则 2）
- [ ] 符合目标刊的字数/格式/引用样式全部 spec？（第 0 层）

**总闸通过后、投稿前，按顺序跑两道终检：**

**① 引用穷尽核验**：调 `reference-checker` 把整份参考文献逐条核实、清空【待核】、按严重度分级（见规则 5 的「靠后·穷尽终检」）。

**②《审稿人预演》**（比静态清单更狠的验证闸）：
以目标刊 referee 视角通读全文，按这个格式产出——
1. **Summary**：一段话复述本文在做什么。
2. **Strengths**：1–3 条具体优点。
3. **Major Concerns**：按严重度排序，每条给证据、说"为什么这是问题"。
4. **Minor**：可改的表述/呈现问题。
5. **Recommendation**：现实的录用/修改判断（别安慰）。
6. **修改计划**：中文、按优先级排。
守则：**区分"致命设计缺陷"与"可改的表述问题"**；不要因为"它不是另一篇论文"而扣分；缺材料就说明限制、不脑补。（这一步薄壳 `journal-drafting` 会自动触发。）

## 9. 防漂移 / 记忆（Drift Control & Memory）

- **每篇一份「成稿决策日志」**（放该论文文件夹，仿你备忘录链的习惯）：记"目标刊为什么选它、综述母本定了哪篇、contribution 一句话、本篇红线"。长工程最大的敌人是漂移。
- 文档顶部**永久钉死"一句话研究问题"**，每个 checkpoint 都拿正文跟它对一次。
- 重大转向（换刊、换框架、砍章节）**留痕**，别静默改——决策能回溯，才不会绕圈。
