# 期刊起草工作流（薄壳，加载通用母版后执行）

这是一层调用壳。真正的规则书是**《英文期刊起草框架》母版**，跨论文复用。

# 交互契约（最重要，贯穿全程）

**一次只推进一步。每一步都「问 → 等 → 再走」，不要一口气跑完多步。**

- 每步**开头先渲染《进度面板》**，让用户知道现在在哪、下一步是什么。
- 每步先**说清这步要做什么、需要用户提供什么** → **提问** → **停下等用户回答/确认** → 才执行。
- 遇到关键闸（三个 ⛔、骨架确认、journal-adapt 产物确认、每节定稿、总闸/终检），**必须等用户明确说"确认/继续"才往下**，不替用户跳闸。
- 用户没让你连跑时，**做完一步就停，报进度，把球交回给用户**。

# 入口路由（进任何步骤前，最先判断一次）

进流程前先分辨用户要的是哪种，**不确定就问一句**：

- **(甲) 从头起草**（"开始起草""写成期刊论文""按期刊标准整理"…）→ 走下面**完整流程**（第 1 步起）。
- **(乙) 只对已有日文稿做日语优化/整理**（"修改日语""整理日语""日语润色""去中文味"…，手里已有日文草稿、不需要从头搭结构）→ **走「日语优化快速入口」，跳过 Phase 0 与起草全流程**：
  1. 进度面板里 `[0]~[3]`、`[4]~[6]` 全标 ⏭️，只点亮 `[3.5] 日语优化`。
  2. **只读** `references/ja/ja-zh-残留字形チェック.md` + `references/ja/ja-学術文体.md`（外加共享风格层 `references/en-economics-writing-style.md`）；**不**加载整本母版 Phase 0、不搭骨架、不逐节起草。
  3. 直接执行下文**第 3.5 步**（只问轨道：投稿哪刊/修論哪校规程 → 净化层三关 → 文体层逐节优化、逐节确认）。
  4. 跑完即止，把球交回用户。**不要**把用户拖进目标刊/层级现实性/结构母本那套起草闸。
- **边界**：若(乙)途中发现稿子结构本身就不成立（缺章节/逻辑断裂，不只是语言问题）→ 提示用户「这超出日语整理范围，要不要转成完整起草流程？」**等用户决定**，不擅自升级。

> 一句话：**"整理日语" ≠ "从头起草"**。前者直奔第 3.5 步，后者才走 Phase 0 全流程。

---

## 第 1 步：加载母版

本 skill **自带母版规则书**，无需外部依赖。开始前，读取与本 `SKILL.md` 同目录下的 [`template-master-framework.md`](template-master-framework.md)（母版：第 0 层 SPEC + 通用硬规则 1–9 + 三道验证闸）。逐节写作时，再按需读取 `references/` 下的展开文件（写作风格 / 文献宽度扩展 / 稿件纯净度 / 领域约定）。

读到母版后，**以母版的「通用硬规则 1–9」为准执行**，本文件只负责把流程串起来。

## 第 1.5 步：目标档位（隐形判断，最先做）

进 Phase 0 前，先定档位（不确定就问一句：「这篇最终目标是 ① 投期刊，还是 ② 学位论文/修論？」）：

- **期刊投稿档** → 全配置跑。
- **学位论文档** → 按母版「适用档位」表**自动跳过/软化期刊专属环节**：动态写作层 ⏭️、层级现实性判断免、新规性软化、引用核验改轻量、审稿人预演降级为答辩委员视角或跳过、匿名化/投稿体例/字数硬限/复现包 ⏭️（改用学校规程）；**通用骨架（规则 1·2·4·5·7·8·9）照走**。

定档后在《进度面板》标出档位，被跳过的阶段标 ⏭️。这一步是"隐形"的：学位论文档不要让用户被一堆用不上的期刊机制淹没，只呈现该走的部分。

## 第 2 步：先过 Phase 0 闸（最重要）

带用户把母版「第 0 层 SPEC 槽位」逐项填实（学位论文档：「目标」槽填**学校规程 + 审查委员**，不做层级现实性判断）（按交互契约：逐项问、等回答），特别盯死三个 ⛔：
- 目标期刊（具体刊名，不许抽象目标）
- 目标层级现实性（贡献/数据/识别诚实对得上吗，对不上就降档）
- 结构母本 3–5 篇

**三个 ⛔ 没填满，不进入正文写作。** 这是硬闸，不要跳过。

**接缝·是否升级动态写作层**（用 `journal-adapt` skill，判断泛用、对任何论文一致）：
- 同时满足 → 建议调用：① 有明确目标刊；② 能拿到该刊 5–8 篇真论文；③ 英文写作。
  调用后产出 `dynamic_writing_skill.md`，作为**动态层**叠在本母版（**静态层 P4**）之上；它本身就是逐步交互（问目标刊→喂语料→萃取→你过目确认），正好契合本契约。
- 不满足（无明确目标刊 / 凑不齐语料 / 非英文，如学位论文按格式规程写）→ ⏭️ 跳过，面板标"不适用"，只用母版静态层。
- **是否调用先问用户、等确认**，不自作主张。

## 第 2.5 步：文献宽度扩展（可选 — 先行研究宽度不足时）

**触发条件**：基础研究/机制已立，但**锚点不到 10 篇**、写先行研究/文献综述时宽度感觉不够。不缺宽度则 ⏭️ 跳过。

按 `references/en-literature-breadth.md` 走（gated）：
1. **先问用户**：「请把你目前的基础锚点论文给我——最核心的那不到 10 篇，标题或 DOI 都行。」**等用户给** → 才开跑。
2. 用 **Semantic Scholar 引用图谱 API**（WebFetch GET，详见参考文件）从锚点做四向滚雪球：向后(references) / 向前(citations) / 推荐(recommendations) / 共被引(多锚点取交集)。
3. 按「相关度 × 影响力 × 新近度」排序，**按簇归位**（测量/识别/机制/方法）。
4. 交给用户一张**按簇分组、簇内排序**的候选清单（状态默认 `待读·待核`）。
5. 用户选定的 → 进 `reference-checker` 核实 → 喂规则 2「文献地图」。

护栏：只列 API 真实返回的论文、绝不补造；定向宽度不注水；英文/国际覆盖好、中日文稀疏暂不强求。

## 第 3 步：搭骨架（要用户确认）

按规则 1 + 选定母本（或动态写作层）搭章节空骨架，每节写一句"做什么、得什么"。**把骨架给用户过目、等确认或调整**，再进逐节写。

## 图表蓝图（Exhibit Plan）（进度面板 `[2.5]`；在第 3 步骨架之后、第 4 步逐节写之前）

**gated**：有结果图表的论文（实证/结构/SCM…）走这步；纯理论命题型 → 软化（exhibit 即命题）。**学位论文档**：蓝图照做，house-style 跟学校規程·目标语言、渲染工具可选（见母版「适用档位」）。**非结果型 ⏭️ 跳过**，面板标"不适用"。

按规则 6「图表先行」+ `references/en-exhibit-design-narrative.md`，**骨架定了、逐节写之前**，先把整套图表当骨干设计出来（沿用《交互契约》问→等→执行）：

1. **先问用户**：「把你要用的结果/数据给我——哪些回归/图、各想说什么。」**等用户给** → 才开跑。
2. **接 journal-adapt**（若已产 `dynamic_writing_skill.md`/`style_profile.md`）：读其治理方法族的 `results_main_object`/`table_presentation` 给蓝图**播种**——该刊这类论文谁领头、系数表怎么走列。没有则按方法族默认（缩减式/gravity→表领头；SCM/事件研究→图领头）。
3. **填《图表蓝图》**（模板见标准文档）：exhibit 清单（编号|类型|承载的唯一论点|锚定节|主/附|领头对象）+ 叙事弧 + **头牌指定**（1–2 张=这篇论文本身）+ 主/附放置。
4. **锁全文 house-style**：投稿跟目标刊（三线表/星号/SE/表注/题注缩写）；学位论文跟規程。
5. **渲染：按环境三选一**（产出 exhibit 时执行，先看手头有什么工具）：
   - **有 Stata** → 调 `scripts/exhibit_style.do`（`exhibit_table` 三线表）+ `scripts/exhibit_figs.do`（`s1mono` 灰度图 / `exhibit_coefplot`）；真实回归先 `eststo` 存好。
   - **无 Stata、有 Python**（`pandas`＋`matplotlib`，不需要 Stata/LaTeX）→ 调 `scripts/exhibit_render.py`：`render_table`（booktabs 三线表片段）/ `render_coefplot` / `render_event_study`，house-style 与 .do 对齐。先 `python exhibit_render.py --demo` 看产物长相。
   - **两者都没有** → 退化为**三线表 Markdown 模板**手工填（顶/中/底三条横线、禁竖线/九宫格、星号三级、SE 在系数下方括号），再逐项过下方**视觉自检清单**。
   ⛔ 三条路径**都只格式化真实估计**，绝不编系数/SE/N/p（规则 5/6）。
6. **把蓝图交用户过目、等确认** → 再进逐节写；逐节写即按此蓝图 write-to-exhibit。

**视觉自检清单**（渲染后亲眼看一遍才算定稿，规则 6/8）：
- [ ] 是**三线表**吗？只有顶/中/底三条横线，**无竖线、无全框线、无九宫格**？
- [ ] 星号三级一致（`* p<0.10, ** p<0.05, *** p<0.01`）、SE 在系数正下方括号？
- [ ] 小数位、列号、表注位置全文统一？图为灰度极简、题注在图下？
- [ ] 表/图自含（标题、单位、样本、注释齐全），脱离正文也看得懂？

护栏：**渲染≠造数**——只格式化真实估计，绝不编系数/SE/N/p（规则 5/6）。

## 第 4 步：逐节写（一节一停）

**一次只写一节**，写完就停、报进度、等用户决定下一节。每节走：
1. 出**正文草稿**（英文，或学位论文轨用对应语言）
2. 出**中文映射解释（write-to-exhibit）**：先对到 `[2.5]` 图表蓝图里的哪张 exhibit / 哪列哪面板，散文走读者穿过它（不是写到哪就地塞张表）；每个主张绑定到具体单元格
3. 出 **Warnings**：无支撑的解读、over-claim、待核引用、组间显著性误述
4. 跑母版《单节自查》，更新该节「匹配分 _/5」

**按所写部分，调母版对应规则块当 checklist：**
- 识别章 → 规则 4「识别审计清单 + 设计专用补检」
- 文献章 → 规则 2「文献地图表」
- 结果章/任何带图表的节 → 规则 6「图表先行 + 表格转写纪律」+ `references/en-exhibit-design-narrative.md`；核对与 `[2.5]` 图表蓝图一致，并过**视觉自检**（渲染成图→看：三线表/无九宫格）
- 任何正文 → 写作风格层 `references/en-economics-writing-style.md`

全程守「主张强度 ≤ 识别强度」（规则 4）、引用诚信（规则 5，不编造、拿不准标【待核】）、对照顶部「一句话问题」防漂移（规则 9）。

## 第 3.5 步：日语优化（仅日语轨 — gated）

**触发条件**：第 0 层「写作语言」槽位 = 日文（投稿日本期刊 / 修論）。**非日语轨 ⏭️ 跳过**，面板标"不适用"。
针对的痛点：以中文构思打底、转写成日文后残留的「中文味」（字形/词汇/直译腔），以及日语学术文体（投稿轨/修論轨两套）不到位。

**时机**（看从哪个入口到这步）：
- 经**完整起草流程**走到这步 → 默认**全篇逐节写完后集中跑**（第 4 步逐节写完成、第 5 步总闸之前）。
- 经**「日语优化快速入口」(乙)** 直接进入 → **立即**对用户已有的日文稿跑（无逐节起草可等）。

两种入口都**不默认逐节、不每次问时机**；用户主动要求每节即跑才逐节。沿用「问 → 等 → 执行」节奏，对**已写好的日文稿**优化：

1. **先问用户**（只问轨道，不问时机）：「这篇日语轨是 ① 投期刊（哪本刊？决定标点二分与体例）还是 ② 修論（哪校规程？）？」**等回答** → 才开跑。
2. **净化层**（按 `references/ja/ja-zh-残留字形チェック.md`，三关顺序）：
   - 第①关 **字形**：跑 `python skills/journal-drafting/scripts/zh_glyph_check.py <节文本>`，清中文简/繁残留字；**引用题名/中国人名地名按原文保留**，`--fix` 后人工复核这些区。
   - 第②关 **词汇/术语**：拿对照表逐段扫，重点盯 ⚠️ 同形异义陷阱（货币政策→金融政策、显著→有意、出口→輸出…）与中式官腔。
   - 第③关 **表达/语序/标点**：直译腔→地道搭配、删范畴词冗余、统一标点二分（，．/、。按目标刊）与书名号『』「」。
3. **文体层**（按 `references/ja/ja-学術文体.md`）：先确认轨道（投稿/修論），再按该轨规范逐节优化——である体・自称（忌「私」入正文）・ヘッジ・引用「著者（年）」・结构定式（投稿はじめに构成预告 / 修論序章+各章概要+終章総括）。
4. 每节出 **优化前后对照 + 改动说明**（属第②③关哪类、为何改），更新该节「匹配分 _/5」。**等用户确认该节** → 下一节。

护栏：本步只改语言表层与文体，**不动论证/数据/引用内容**；拿不准是否该保留某字/某专名 → 标记问用户，不擅改。

## 第 5 步：齐稿终检（三道，逐道停）

1. **总闸**：过母版《总闸》清单。
2. **引用穷尽核验**：调 `reference-checker` skill 把整份参考文献逐条核验、清空所有【待核】、按严重度分级（地基引用应在文献章动笔前就已早查过一遍，见规则 5）。
3. **审稿人预演**：以目标刊 referee 视角跑母版《审稿人预演》——Summary / Strengths / Major Concerns（按严重度、给证据）/ Minor / 现实 Recommendation / 中文优先级修改计划。区分致命设计缺陷 vs 可改表述问题，缺材料就说明、不脑补。

## 边界

- 不编造引用、数据、新规性证据、机构事实。拿不准先问、先标【待核】。
- 本壳只串流程、管交互节奏；规则冲突时以母版为准。
