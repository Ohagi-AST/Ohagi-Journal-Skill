# 期刊起草工作流（薄壳，加载通用母版后执行）

这是一层调用壳。真正的规则书是**《英文期刊起草框架》母版**，跨论文复用。

---

# 交互契约（最重要，贯穿全程）

**一次只推进一步。每一步都「问 → 等 → 再走」，不要一口气跑完多步。**

- 每步**开头先渲染《进度面板》**，让用户知道现在在哪、下一步是什么。
- 每步先**说清这步要做什么、需要用户提供什么** → **提问** → **停下等用户回答/确认** → 才执行。
- 遇到关键闸（三个 ⛔、骨架确认、journal-adapt 产物确认、每节定稿、总闸/终检），**必须等用户明确说"确认/继续"才往下**，不替用户跳闸。
- 用户没让你连跑时，**做完一步就停，报进度，把球交回给用户**。

## 《进度面板》模板（每步开头渲染一次）

```
📋 起草进度 —— [论文代号] ｜ 档位：期刊投稿 / 学位论文
[0] SPEC 第0层        ✅/🔄/⬜
[0.5] 文献宽度扩展(可选) ✅/🔄/⬜/⏭️
[1] 动态写作层(可选)  ✅/🔄/⬜/⏭️
[2] 骨架              ✅/🔄/⬜
[3] 逐节写：
    - Abstract        草稿✓ 自查✓ 匹配分 _/5
    - Introduction    🔄 ...
    - ...             ⬜
[4] 总闸              ⬜
[5] 引用穷尽核验      ⬜
[6] 审稿人预演        ⬜
→ 现在这步：____    下一步：____
```
图例：✅完成 🔄进行中 ⬜未开始 ⏭️跳过(不适用)。「匹配分 _/5」=该节像不像目标刊（1 很不像、5 很契合），随修改升降，给用户进度体感。

## 第 1 步：加载母版

本 skill **自带母版规则书**，无需外部依赖。开始前，读取与本 `SKILL.md` 同目录下的 [`起草框架母版.md`](起草框架母版.md)（母版：第 0 层 SPEC + 通用硬规则 1–9 + 三道验证闸）。逐节写作时，再按需读取 `references/` 下的展开文件（写作风格 / 文献宽度扩展 / 稿件纯净度 / 领域约定）。

读到母版后，**以母版的「通用硬规则 1–9」为准执行**，本文件只负责把流程串起来。

## 第 1.5 步：目标档位（隐形判断，最先做）

进 Phase 0 前，先定档位（不确定就问一句：「这篇最终目标是 ① 投期刊，还是 ② 学位论文/修論？」）：

- **期刊投稿档** → 全配置跑。
- **学位论文档** → 按母版「适用档位」表**自动跳过/软化期刊专属环节**：动态写作层 ⏭️、层级现实性判断免、新规性软化、引用核验改轻量、审稿人预演降级为答辩委员视角或跳过、匿名化/投稿体例/字数硬限/复现包 ⏭️（改用学校规程）；**通用骨架（规则 1·2·4·5·7·8·9）照走**。

定档后在《进度面板》标出档位，被跳过的阶段标 ⏭️。这一步是"隐形"的：学位论文档不要让用户被一堆用不上的期刊机制淹没，只呈现该走的部分。

## 第 2 步：先过 Phase 0 闸（最重要）

带用户把母版「第 0 层 SPEC 槽位」逐项填实（学位论文档：「目标」槽填**学校规程 + 审查委员**，不做层级现实性判断）（按交互契约：逐项问、等回答），特别盯死三个 ⛔：
- 目标期刊（具体刊名，不许抽象目标）
- 目标层级现实性（贡献/数据/识别诚实对得上吗，对不上就降档）
- 结构母本 3–5 篇

**三个 ⛔ 没填满，不进入正文写作。** 这是硬闸，不要跳过。

**接缝·是否升级动态写作层**（用 `journal-adapt` skill，判断泛用、对任何论文一致）：
- 同时满足 → 建议调用：① 有明确目标刊；② 能拿到该刊 5–8 篇真论文；③ 英文写作。
  调用后产出 `dynamic_writing_skill.md`，作为**动态层**叠在本母版（**静态层 P4**）之上；它本身就是逐步交互（问目标刊→喂语料→萃取→你过目确认），正好契合本契约。
- 不满足（无明确目标刊 / 凑不齐语料 / 非英文，如学位论文按格式规程写）→ ⏭️ 跳过，面板标"不适用"，只用母版静态层。
- **是否调用先问用户、等确认**，不自作主张。

## 第 2.5 步：文献宽度扩展（可选 — 先行研究宽度不足时）

**触发条件**：基础研究/机制已立，但**锚点不到 10 篇**、写先行研究/文献综述时宽度感觉不够。不缺宽度则 ⏭️ 跳过。

按 `references/文献宽度扩展.md` 走（gated）：
1. **先问用户**：「请把你目前的基础锚点论文给我——最核心的那不到 10 篇，标题或 DOI 都行。」**等用户给** → 才开跑。
2. 用 **Semantic Scholar 引用图谱 API**（WebFetch GET，详见参考文件）从锚点做四向滚雪球：向后(references) / 向前(citations) / 推荐(recommendations) / 共被引(多锚点取交集)。
3. 按「相关度 × 影响力 × 新近度」排序，**按簇归位**（测量/识别/机制/方法）。
4. 交给用户一张**按簇分组、簇内排序**的候选清单（状态默认 `待读·待核`）。
5. 用户选定的 → 进 `reference-checker` 核实 → 喂规则 2「文献地图」。

护栏：只列 API 真实返回的论文、绝不补造；定向宽度不注水；英文/国际覆盖好、中日文稀疏暂不强求。

## 第 3 步：搭骨架（要用户确认）

按规则 1 + 选定母本（或动态写作层）搭章节空骨架，每节写一句"做什么、得什么"。**把骨架给用户过目、等确认或调整**，再进逐节写。

## 第 4 步：逐节写（一节一停）

**一次只写一节**，写完就停、报进度、等用户决定下一节。每节走：
1. 出**正文草稿**（英文，或学位论文轨用对应语言）
2. 出**中文映射解释**：每个主张对到表格/证据的哪一处
3. 出 **Warnings**：无支撑的解读、over-claim、待核引用、组间显著性误述
4. 跑母版《单节自查》，更新该节「匹配分 _/5」

**按所写部分，调母版对应规则块当 checklist：**
- 识别章 → 规则 4「识别审计清单 + 设计专用补检」
- 文献章 → 规则 2「文献地图表」
- 结果章 → 规则 6「表格转写纪律」
- 任何正文 → 写作风格层 `references/经济学写作风格.md`

全程守「主张强度 ≤ 识别强度」（规则 4）、引用诚信（规则 5，不编造、拿不准标【待核】）、对照顶部「一句话问题」防漂移（规则 9）。

## 第 5 步：齐稿终检（三道，逐道停）

1. **总闸**：过母版《总闸》清单。
2. **引用穷尽核验**：调 `reference-checker` skill 把整份参考文献逐条核验、清空所有【待核】、按严重度分级（地基引用应在文献章动笔前就已早查过一遍，见规则 5）。
3. **审稿人预演**：以目标刊 referee 视角跑母版《审稿人预演》——Summary / Strengths / Major Concerns（按严重度、给证据）/ Minor / 现实 Recommendation / 中文优先级修改计划。区分致命设计缺陷 vs 可改表述问题，缺材料就说明、不脑补。

## 边界

- 不编造引用、数据、新规性证据、机构事实。拿不准先问、先标【待核】。
- 本壳只串流程、管交互节奏；规则冲突时以母版为准。
