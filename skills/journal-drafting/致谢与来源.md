# 致谢与来源（Credits & Provenance）

本套「论文写作框架」（母版 + 风格层 + 三个 skill）在搭建过程中，学习、借鉴或直接采用了以下开源仓库。
记录于此：一是**致谢**这些作者的工作，二是**备查**——日后要追溯某条规则从哪来、或想回去看原始设计时，从这里出发。

> 使用边界：我们提取的是**方法论、结构、清单与写作纪律**，不复制其私有内容；采用整份 skill 的，按其许可证使用并保留出处。

---

## 1. affaan-m/ecc

- **仓库**：https://github.com/affaan-m/ecc
- **是什么**：一套把 Claude Code 当"agent harness 操作系统"经营的完整体系（agents / skills / rules / hooks / memory）。
- **我们学到/用了什么**：**整体战略心智模型**——把工作拆成 Rules（硬约束）/ Skills（可复用工作流）/ 验证闸 / Memory 四件套；"研究优先（动手前先穷尽调研复用）""验证闭环，别 hope-driven""记忆持久化防漂移"。我们母版的「Rules-as-spec + Verification-loop + Memory」骨架来自这里。
- **采用形式**：仅借鉴思想，未拷贝文件。
- **许可证**：MIT。

## 2. lishn6/awesome-ai-econ-research-writing

- **仓库**：https://github.com/lishn6/awesome-ai-econ-research-writing
- **是什么**：面向经济学论文的一组写作 skill（economics-writing-style、identification-auditor、literature-positioning、referee-simulator、regression-table-writer、good-question、empirical_workflow 等）。
- **我们学到/用了什么**：把抽象规则**落成可操作清单**。具体吸收进母版——
  - `references/经济学写作风格.md` ← economics-writing-style（信息密度、禁套话、用词 avoid→prefer 表、因果谨慎句库、数字/图表注释）
  - 规则 1 各节配方、"固定效应≠识别" ← empirical_workflow
  - 规则 2 文献地图表 ← literature-positioning
  - 规则 4 识别审计清单 + 设计专用补检 + 因果措辞替换库 ← identification-auditor
  - 规则 6 表格转写纪律 ← regression-table-writer
  - 规则 8《审稿人预演》第三道闸 ← referee-simulator
- **采用形式**：提炼其方法与清单，改写为中文母版内容；未整份安装。
- **许可证**：见该仓库 LICENSE。
- **二级出处**：其 good-question 标注灵感来自 Rimagination/good-question。

## 3. Liuxiangjian-ai/reference-checker-skill

- **仓库**：https://github.com/Liuxiangjian-ai/reference-checker-skill
- **是什么**：投稿前参考文献穷尽核验 skill（英文走 DOI/Crossref/PubMed，中文走 CNKI/万方/VIP，分 Verified/Minor/Major/Critical）。
- **我们学到/用了什么**：作为**整套 skill 直接安装使用**，接为终检第 2 道（引用穷尽核验），并在母版规则 5 设"地基引用早查 + 投稿前穷尽终检"双时点。
- **采用形式**：原样安装于 `~/.claude/skills/reference-checker/`。
- **许可证**：见该仓库 LICENSE。

## 4. WantongC/journal-adapt-writing-skill

- **仓库**：https://github.com/WantongC/journal-adapt-writing-skill
- **是什么**：从 5–8 篇目标刊真论文里**萃取该刊写作文化**、生成"单刊单稿专用动态写作 skill"的两阶段框架（静态层 + 动态层、五级优先级、反抄袭边界、人工闸）。
- **我们学到/用了什么**：① 作为**整套 skill 直接安装**，当"动态写作层"引擎；② 借鉴其**gated 交互**（问→喂语料→萃取→过目确认）补进我们 `journal-drafting` 薄壳的《交互契约》+《进度面板》；③ 用它的"静态层 P4 可外挂"设计，把我们母版接为它的静态层。
- **采用形式**：原样安装于 `~/.claude/skills/journal-adapt/`；其 base_rules、docs 一并保留。
- **许可证**：MIT。
- **二级出处**：其 README 推荐的静态写作 skill（hanlulong/econ-writing-skill、Master-cai/Research-Paper-Writing-Skills、SNL-UCSB/paper-writing-skill、lishix520/academic-paper-skills、blader/humanizer）我们暂未采用，列此备查。

## 5. Lambenthan/empiricalwiki（及其上游 ΩmegaWiki、Karpathy LLM-Wiki）

- **仓库**：https://github.com/Lambenthan/empiricalwiki ；上游 https://github.com/skyllwt/OmegaWiki ；理念源 Karpathy LLM-Wiki gist。
- **是什么**：以 wiki/知识图谱为核心的经管实证研究操作系统（把论文拆成 variables/identification/… 等实体、跨论文累积、29 个 skill 覆盖全流程）。整套基建偏重单篇、已收尾型的实证项目，故**未采用整套**。
- **我们学到/用了什么**：仅借鉴其 `/discover` skill 的**引用图谱滚雪球方法**——从锚点论文向后/向前/共被引/横向扩展文献宽度。落成 `references/文献宽度扩展.md`，并接入薄壳「第 2.5 步 文献宽度扩展」与母版规则 2。用 Semantic Scholar Graph API 实现，不装其基建。
- **采用形式**：方法借鉴，未安装。
- **许可证**：MIT（empiricalwiki / ΩmegaWiki）。
- **配套数据源**：Semantic Scholar Graph API（免费，引用图谱）。

## 6. juliaError/econ-TopJournal-writing-Skill

- **仓库**：https://github.com/juliaError/econ-TopJournal-writing-Skill
- **是什么**：经济学顶刊写作 5-skill 系统（英文写作 / 中文顶刊 / 表图设计 / 多代理 / 路由），`econ-write` 基于 hanlulong/econ-writing-skill（MIT），蒸馏自 50+ 篇顶刊写作指南。
- **⚠️ 许可证**：原创内容 **CC BY-NC 4.0（非商用）**，作者声明不以 OSI 开源形式发布 → **不可复制/转载进发布仓库**，只能**理解后用自己的话再造 + 署名引用**。
- **我们学到/用了什么（均为再造，未复制文字）**：
  - `references/稿件语气与正文纯净度.md`（三通道分离 + 段级 linter + 指针/硬拒）← 其 manuscript-voice 思路
  - 母版规则 6「系数→经济量级」← 其 economic-magnitude 思路
  - `references/领域写作约定.md`（含贸易/引力等子领域）← 其 field-specific conventions 思路
- **刻意未采用**：表格/图形视觉设计（已有 build 脚本）、多代理控制器（现阶段过重）——记此备查。
- **采用形式**：思路借鉴 + 自行再造，未安装、未复制。

## 7. 日语优化层的语料来源（日本語学術コーパス）

- **是什么**：`references/中文残留チェック.md`（净化层）与 `references/日本語学術文体.md`（投稿轨/修論轨文体层）的文体规则，蒸馏自一批公开的日本经济学学术文本。
- **语料来源**（均为公开获取，仅用于**蒸馏文体规律**，不复制原文进发布物）：
  - **J-STAGE** 开放获取经济学/社会科学期刊论文（投稿轨主体，按刊过滤书评/学会公告后取样）。
  - **一橋大学『経済研究』** OAI-PMH 开放仓库（投稿轨补充）。
  - **上智大学・明治大学** 经济学博士/学位论文（修論轨，经 NDL 等公开渠道）。
- **我们学到/用了什么**：である体・自称纪律・ヘッジ・引用「著者（年）」・标点二分・投稿与修論两轨结构差异等**文体规律**，提炼为中文规则文件；中文残留字形表（`scripts/zh_glyph_check.py` 的 ZH2JP）为自建。
- **采用形式**：语料只读用于观察归纳，**未将原文随包发布**；采集脚手架（下载/转写/抽样脚本与原始 PDF）仅留本地、不进仓库（见根 `.gitignore` 的 `corpus_build/`）。
- **许可证/边界**：尊重各刊与各库的开放获取条款；本层产出为对**公开文体惯例**的再表述，不含受限的私有内容。
