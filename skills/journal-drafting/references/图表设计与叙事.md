# 图表设计与叙事（全文级·图表先行层）

> 母版**规则 6** 的展开，与 `经济学写作风格.md §图表`（句级 caption 标准）分工：**本文件管全局**——
> 整套图表怎么**先行设计、承载论点、串成叙事、统一规格、定位主附、视觉验收**。
> 经验基线来自对目标刊真实论文的视觉+结构侦察（本套以 **EER 全 10 篇**为基线；换刊换基线，方法不变）。
> 语言无关：英文刊与日文修論/中文毕业论文都适用，差异见末节「适用档位」。

## 核心理念：段落服务图表，图表服务叙事

论文是给人读的；读者最先看、最常回看的是图表。所以顺序是 **先定图表 → 散文服从图表 → 图表服从叙事**，不是"写到哪、提一句、塞张表"。

- ⛔ **反模式（本层要消灭的四宗罪）**：
  - **不统一**：每张表各写各的格式 → 用全文一套 house-style（下文 §3）。
  - **不显眼**：没有"头牌" → 指定 1–2 张"就是这篇论文"的核心图表（§2）。
  - **位置乱**：提到就地塞 → 按叙事弧与主/附计划放置（§2）。
  - **提到就放**：prose-first 倒推 → write-to-exhibit，先有图表再写散文（§4）。

## 1. 图表蓝图（Exhibit Plan）——动笔正文前先填

仿母版第 0 层 SPEC：每开一篇，复制下面这块填实，**经用户确认后才进逐节写**。

```
论文代号：________   方法族（缩减式/gravity｜SCM/事件研究｜结构-校准｜理论…）：________
头牌图表（1–2 张 = 这篇论文本身）：________

# Exhibit 清单（先列全、再写散文）
编号 | 类型(表/图) | 承载的唯一论点（一句话） | 锚定节 | 主文/附录 | 领头对象
T1   | 表          | 样本与变量描述           | Data   | 主文      | 描述统计表
...  |             |                          |        |           |
F1   | 图          | 动机事实/趋势            | Intro  | 主文      | 趋势图
...  |             |                          |        |           |

# 叙事弧（exhibit 串成论证，每张是一个 beat）
动机图 → 描述表 → 基准结果(走列) → 扩展/分解 → 稳健 → 机制/异质 → [附录]
```

**叙事弧（EER 全 10 篇高度一致，作模板）**：
1. **动机图 Fig.1**（Intro）：一张刺激读者的趋势/事实图。
2. **描述统计 Table 1**（Data）：样本·变量·口径。
3. **基准主结果**（Results 开头）：**走列**从约束到丰富设定。
4. **扩展 / 分解 / 机制**：逐张递进。
5. **稳健性**：靠后，常直接命名 "Robustness checks"。
6. **异质性 / 机制图**。
7. **附录**：placebo/falsification、全样本估计、数据来源、额外稳健。

**头牌按方法族定（不可一刀切，实证自 EER）**：
- 缩减式 / gravity / PPML / IV / 结构分解 → 头牌 = **基准系数表**（走列）。
- SCM / 合成控制 / 事件研究 / SDID → 头牌 = **主事件研究/合成控制图**。
- 理论 → 头牌 = **核心命题/推导**。
- ⛔ 接 journal-adapt：若已萃取目标刊 `results_main_object`/`table_presentation`，**以该刊治理方法族的领头方式为准**。

**主/附放置（EER ≈ 正文 76% / 附录 24%）**：
- **主结果与基准识别一律留正文**（接 journal-adapt 硬规则：系数表不得流放附录）。
- 附录承接：placebo/falsification、全样本/全系数、数据来源与构造、额外稳健。
- 附录 exhibit 用字母前缀（Table A.1 / Fig. B.7）。

## 2. 全文统一 house-style（一套规格管全文所有 exhibit）

> 句级 caption / 表注细则见 `经济学写作风格.md §图表`，此处不重复；本节锁**全文一致性**。

**表（实证自 EER，全部如此）**：
- **三线表**（booktabs：`\toprule`/`\midrule`/`\bottomrule`）。⛔ **禁竖线、禁全框线、禁九宫格。**
- 跨列被解释变量头（`mgroups` → `\multicolumn`+`\cmidrule`）；列**编号 (1)(2)…** 走列。
- 系数带显著性星号；**标准误在系数正下方括号内**；系数小数位全文一致（EER 约 3 位）。
- **FE 指示行**（Year FE / Unit FE… = Yes/No 文字）；底部 `Observations`（千分位）、`R²` 等统计行。
- **表注**段在表下：方法 → 样本 → 关键变量定义 → 聚类 → 显著性约定。
- 显著性约定（EER）：`* p<0.10, ** p<0.05, *** p<0.01`（三级）。
- **题注在表上方**：`Table N` + 句子式标题。

**图（实证自 EER，全部如此）**：
- **灰度/黑白为主**，print-friendly；地理异质性才用低饱和 choropleth。⛔ 无 3D、无背景填充、无 chartjunk、网格线极淡或无。
- **题注在图下方**：`Fig. N`（Elsevier 缩写，非 `Figure`）+ 句子式标题 + `Note:` 行（方法/CI 定义/来源）。
- 主力图型：系数图/毛毛虫图（点+CI，常按估计值排序或分面）、合成控制 actual-vs-synthetic、placebo gap 图、事件研究图（点+CI+零线）、核密度、choropleth。
- 字体与正文一致；轴标签字号克制。

**渲染工具（落上面规格、且不造数）**：见 `scripts/`——`exhibit_style.do`（esttab→booktabs 三线表）、`exhibit_figs.do`（s1mono 黑白 + coefplot）、`exhibit_preamble.tex`。题注缩写跟刊（EER=`Fig.`），换刊改这一处。

## 3. write-to-exhibit 纪律 + 引出句式库

**先有 exhibit、散文走读者"穿过"它**（保留母版规则 6"每句绑定到一个表格列/面板/行"，只是把方向倒过来：从图表出发写散文，而非写到哪塞张表）。

**引出句式库（取自 EER 全 10 篇正文，英文轨直接复用）**：
- **呈现/落位**：`We present results in Table N.` ／ `Our main results are presented in Table N.` ／ `Table N presents/reports/displays …` ／ `We report the results in Table N.`
- **走列**：`Column (1) of Table N replicates …` ／ `The second column of Table N presents …` ／ `Columns 1 and 3 of Table N show …`
- **解读/推断**：`Table N shows that …` ／ `The results in Table N indicate/imply that …` ／ `An overarching implication of Table N is that …`
- **跨图表衔接（构成逻辑链）**：`… in line with those reported in Table M` ／ `repeat the baseline results from Table M, column k`
- **图**：`Fig. N illustrates/presents/plots/shows …`
- ❌ 空洞："Table X is shown below." / "见表 X"（不说它干什么、和上下文什么关系）。

## 4. 视觉自检硬闸（杜绝九宫格这类低级错误）

> 文字流看不见三线表 vs 九宫格；**必须把图表渲染成图、亲眼看一遍**。

凡产出/更新图表的节，定稿前过这道闸：

1. 渲染：`scripts/exhibit_*` 出 `.tex`/图 → latexmk 编译 → `python scripts/pdf_to_png.py out.pdf` 成 PNG。
2. 看 PNG，逐条核：
   - [ ] 是**三线表**吗？有没有混进竖线/全框线/九宫格？
   - [ ] 列对齐、小数位、星号、SE 位置符合 house-style？
   - [ ] 题注（表上/图下）、表注（方法/样本/聚类/显著性）齐全？
   - [ ] 和图表蓝图里这张的"承载论点/领头方式"一致？
3. 不达标 → 改，重渲染重看，直到过。

## 5. 渲染≠造数（接母版规则 5/6，红线）

- 渲染工具只是**真实 `eststo`/`estimates store` 估计的格式化器**；系数/标准误/N/p **一律来自真实回归**。
- ⛔ 绝不手写/编造任何数字进 `.tex`；缺值标【待核】或问用户、从数据算。
- 图里画的是真实估计/真实数据点，不编造点位与区间。

## 6. 适用档位：期刊投稿 vs 学位论文（修論/毕业论文）

**不是整层软化**——本层绝大部分对学位论文同样适用、部分更需要。区分如下：

**照走（学位论文同样需要，部分更需要）**
- 图表先行哲学、图表蓝图规划（清单/锚定章节/叙事弧/头牌按方法族）。
- **三线表铁律 + 视觉自检**——学生最易犯九宫格，这两条对学位论文**价值更高**。
- 渲染≠造数；write-to-exhibit 原则。

**弱化/替换（期刊·英文·LaTeX 专属，按学位论文自身轨替换，不硬套 EER）**
- **house-style「跟谁」**：投稿跟目标刊；学位论文跟**学校規程 / 目标语言**——题注缩写（`Fig.`／`図`／`表`）、星号约定、字号、引用样式都从規程。
- **引出句式库**：英文 EER 句式仅英文轨；日文修論/中文毕业论文用**对应语言**的引出惯例。
- **主/附比例**：EER 76/24 仅参考；学位论文按院系規程，原则（主结果与基准识别留正文）不变、比例不锁。
- **渲染工具链**：LaTeX 轨用 `exhibit_*`（esttab→booktabs）；**Word 轨**（多数修論/毕业论文）把 house-style 当规格手工套，或 `esttab using x.rtf` 出 Word 表——工具可选、三线表原则不变。
