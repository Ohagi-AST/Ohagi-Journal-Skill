# 投稿相关工作流（审查、缩稿、R&R）

> 用途：服务入口 F，也可在完整成稿流程终检时加载。只处理论文与投稿相关任务，不扩展到基金、演讲、survey、policy brief 或 op-ed。

## 目录

1. 投稿任务卡
2. 快速审查
3. 深度审查
4. Working paper → journal submission
5. R&R / referee response
6. 官方要求核验
7. 输出与升级边界

## 1. 投稿任务卡

开始前只收任务所需最小事实：

```
任务（快速审查 / 深度审查 / 缩稿 / R&R）：________
目标期刊与当前阶段：________
输入材料（全文/章节/编辑信/审稿意见）：________
作者已真实完成的修改或新增分析：________
允许的改动范围：________
```

材料不足仍可做结构诊断；任何具体结果、页码、行号、表号或“已完成修改”缺失时，必须用 `【作者补】`，不得编造。

## 2. 快速审查

先给 5–10 分钟可执行扫描：

1. 非本领域经济学者能否复述问题、方法和主结果？
2. contribution 是否扛得住最近邻？
3. estimand、识别变异和主张上限是否一致？
4. 头牌图表是否承载主结果，摘要/引言/结论是否与之对齐？
5. 是否有编造/待核引用、隐去零结果、无正式检验的组间比较？

输出前三项高杠杆修复，再列次要问题；不要给 Top-5 readiness 总分。

## 3. 深度审查

用三种视角分别审查，最后合并去重：

- **Methodologist**：estimand、识别假设、推断、稳健性、功效与复现。
- **Field Expert**：问题价值、最近邻、机制、经济量级、外部有效性。
- **Writing Critic**：信息顺序、段落角色、图表自含、语气、重复与内部工序语。

每条问题必须含 `Blocker/Major/Minor + 证据位置 + 为什么重要 + 如何修`。区分设计缺陷、缺材料和纯表达问题，不因“可以写另一篇论文”而扣分。

## 4. Working paper → journal submission

1. 核验目标刊当前官方篇幅、exhibit、匿名、附录、数据与披露要求。
2. 写一句“不可删除的核心贡献”，据此标主文核心、可压缩、可移附录、应删除。
3. 默认裁剪顺序：重复动机 → 文献旁支 → 不回应威胁的稳健性 → 重述已知理论 → 冗长工序说明；不先砍识别与主结果。
4. 核心 estimand、识别逻辑和主要 exhibit 留主文；附录项目仍由主文准确引用。
5. 输出逐节预算和迁移表，不直接按一个历史页数目标硬切；若目标刊无硬限，仍以读者负担和真实母本决定。

## 5. R&R / referee response

- 按“编辑意见 → Referee 1/2… → 原评论 → 回应 → 修改位置”逐点组织。
- 每条回应说明：理解的问题、实际采取的动作、真实新证据、稿件位置、结论如何改变。
- 同意时直接承认并说明修复；不同意时先准确复述担忧，再用设计、证据或范围边界解释，不写防御性套话。
- 新分析只写作者已完成并提供的内容；未完成则写计划/占位，不能生成系数、F 统计量、页码或表号。
- 交叉影响要同步更新摘要、引言、结果、结论、图表和附录；response 与 revised manuscript 必须一致。

## 6. 官方要求核验

涉及页数、字数、匿名、文件拆分、数据包、AI 披露或引用格式时：

1. 优先目标期刊官方 author guidelines / editorial policy / data policy。
2. 记录页面标题、URL、核验日期与适用稿件类型。
3. 搜索摘要、博客或旧 working paper 只能作线索，不能覆盖官方页面。
4. 官方要求与目标刊真实语料不一致时：合规项服从官方要求，风格项用真实语料并标明推断。

## 7. 输出与升级边界

- **快速任务**：交付改写/审查结果 + 改动说明 + Warnings，不展示完整 0A–6 面板。
- **整篇深改**：若需要新增实质主张、重搭全篇或补做分析，说明将升级到入口 A/B，并等用户确认后补完整 SPEC。
- **事实边界**：不替作者声称已做修改、不虚构编辑偏好、不猜目标刊政策。
