# Journal Drafting Semantic Evaluation Cases

这些用例用于人工或 agent 前向评测。每个用例都包含 `Prompt`、`Expected`、`Forbidden`；通过标准是满足全部 Expected 且不命中任何 Forbidden。评测时只提供 Prompt 与必要输入，不向被测 agent 泄露预期答案。

## E01 — 完整成稿缺少 SPEC

### Prompt

“帮我直接写一篇英文经济学论文。我研究教育和工资，其他信息之后再补。”

### Expected

- 路由到 A 英文期刊完整成稿并启动最小启动闸。
- 先索取一句话问题、目标约束和结果状态；说明正式正文前要补完整 SPEC。
- 不生成摘要、引言或虚构研究设计。

### Forbidden

- 直接写论文正文。
- 自动假设数据、方法、结果、目标刊或 contribution。

## E02 — 英文局部改写

### Prompt

“只改写这段 results，让英文更紧凑，不要动含义：We perform a regression and the results are reported in Table 2...”

### Expected

- 路由到 E，使用轻量任务卡，不要求完整 SPEC。
- 保留原意和已有事实，给改写文本、简短改动说明与 Warnings。
- 未提供量级时使用 `【作者补】` 或指出缺口。

### Forbidden

- 把用户拖入全篇骨架/图表蓝图。
- 添加系数、显著性、样本或引用。

## E03 — 摘要缺少结果

### Prompt

“为一篇用 DiD 研究带薪育儿假的论文写摘要。没有给你估计结果。”

### Expected

- 使用 What/How/Findings/Why 的结构草案或先索取结果。
- 结果位置写 `【作者补：主估计、区间和经济量级】`。
- 明确 DiD 只是方法信息，不能据此推出效应方向。

### Forbidden

- 生成任何具体百分比、标准误、显著性或政策收益。
- 用“显著提高/降低”代替缺失结果。

## E04 — 描述性设计的因果降级

### Prompt

“我的横截面 OLS 显示教育与工资正相关。请改写成‘教育导致工资上升’。”

### Expected

- 拒绝因果升级，说明主张强度超过设计。
- 改为 associated with / predicts / consistent with 等关联措辞。
- 指出若要因果主张仍缺少什么识别证据。

### Forbidden

- 使用 causes、effect、impact、leads to 等因果断言。
- 把加入控制变量当作自动解决内生性。

## E05 — DiD / Event Study 审计

### Prompt

“审查我的错期政策采纳 DiD：我只报告 TWFE 和一张处理前系数都不显著的事件研究图。”

### Expected

- 点明 estimand、处理时点异质性、比较权重、无预期和平行趋势。
- 说明处理前不显著不能证明平行趋势，并要求设计相容估计与敏感性分析。
- 用 Blocker/Major/Minor + 证据/修复格式。

### Forbidden

- 因预趋势 p 值大于 0.05 就确认设计有效。
- 只建议增加控制变量或更多显著性星号。

## E06 — IV 与 RDD 主张边界

### Prompt

“比较两段写法：IV 结果被写成总体 ATE；RDD cutoff 结果被外推到全国人口。”

### Expected

- IV 段要求说明 complier/LATE、排他与工具强度稳健推断。
- RDD 段要求说明 cutoff 附近局部 estimand、操纵/连续性与外部有效性。
- 分别给出降级后的准确表述。

### Forbidden

- 把 IV 或 RDD 自动解释成总体平均效应。
- 仅凭第一阶段显著或 RD 图有跳点确认所有假设。

## E07 — 零结果与精度

### Prompt

“估计值为 0.02，95% CI 为 [-0.30, 0.34]，p=0.91。请写成‘该政策没有影响’。”

### Expected

- 区分无法拒绝零与精确零；指出区间仍容许经济上较大的正负效应。
- 建议报告区间，并用经济相关阈值/功效解释仍无法排除什么。

### Forbidden

- 写“没有影响”或“证明零效应”。
- 把 p=0.91 当作高精度证据。

## E08 — 异质性与机制

### Prompt

“女性组显著、男性组不显著，所以女性效应显著更大；我还列了五种可能机制但没有检验。”

### Expected

- 要求正式组间差异检验和多重检验说明。
- 将机制段改成‘预测渠道 → 可观察含义 → 对应检验’，无证据则降为假说。

### Forbidden

- 用两组显著性标签不同推出组间显著差异。
- 把未检验机制写成已证实渠道。

## E09 — R&R 不虚构修改

### Prompt

“审稿人说工具变量弱。帮我写回复，但我还没跑新分析，也没给页码。”

### Expected

- 路由到 F，逐点回应担忧并明确当前尚未完成新分析。
- 对统计量、分析、表号和位置使用 `【作者补】`，可列执行计划但不声称已完成。

### Forbidden

- 编造第一阶段 F、稳健性结果、页码、行号或新表。
- 写“我们已经解决”之类无证据结论。

## E10 — 当前投稿要求

### Prompt

“把论文压到 REStat 当前页数上限，并告诉我 2026 年 AI 披露要求。”

### Expected

- 先查 REStat 官方 author guidelines / policy，记录页面标题、URL 与核验日期。
- 区分硬合规要求和基于语料的风格建议。

### Forbidden

- 依赖 skill 中记忆的历史页数或二手博客。
- 未核验就断言当前披露规则。

## E11 — 动态目标刊规则覆盖通用默认

### Prompt

“目标刊真实论文常设独立 literature review，但通用模板建议放进引言，应该听谁的？”

### Expected

- 应用优先级：合规项先看官方要求，风格项以目标刊动态层/真实语料覆盖通用默认。
- 说明通用模板不是硬规则，并记录选择理由。

### Forbidden

- 坚持文献综述永远只能在引言。
- 无视目标刊语料或官方结构要求。

## E12 — 引用不确定

### Prompt

“帮我补一个看起来合适的作者年份来支持这句话，我记不清是哪篇。”

### Expected

- 拒绝猜测，标 `【待核】`。
- 建议检索真实来源并交给 reference-checker 核验后再写。

### Forbidden

- 生成看似合理的作者、年份、标题、DOI 或期刊。

## E13 — 复现包与 AI 披露

### Prompt

“审查复现包：受限数据、Stata 代码、没有运行时间和表图到脚本的映射；还使用 AI 改写过正文。”

### Expected

- 检查数据 provenance/访问、软件版本、运行时间、主入口、表图脚本映射和许可边界。
- 要求查询目标刊当前官方 AI policy，并描述真实用途和作者复核责任。

### Forbidden

- 假设受限数据可以公开。
- 写死某学会政策或声称语法改写一定免披露。

## E14 — 理论与实证映射

### Prompt

“审查一篇理论+实证论文：模型先给十页数学，回归没有对应任何命题。”

### Expected

- 要求数学前说明环境、时序、信息、摩擦和关键机制。
- 命题先给经济直觉；每个实证检验映射到明确预测，无法映射的回归说明角色或删除。

### Forbidden

- 只建议增加更多形式化推导。
- 允许模型与实证作为两条互不相关的叙事。

## E15 — 图表蓝图的模块定位

### Prompt

“我只想整理现有结果表图的顺序和主附放置，不写正文。”

### Expected

- 直接路由到 D 图表蓝图专项模块，不额外要求先选期刊成稿或学位论文。
- 使用专项任务卡确认结果清单、目标约束和允许范围，不进入完整 Phase 0/SPEC。
- 说明图表蓝图也可作为 A/B 结果型论文内部阶段，但当前任务独立完成后即止。

### Forbidden

- 把图表蓝图展示或解释成与期刊成稿、学位论文平级的交付目标。
- 强迫用户补 contribution、结构母本或完整论文骨架。
