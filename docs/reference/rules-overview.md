# 母版规则概要（Rules Overview）

母版 [`template-master-framework.md`](../../skills/journal-drafting/template-master-framework.md) 的「通用硬规则 1–9」一页速查。**权威表述以母版为准**，本页只为快速定位。

> 设计来源：把 agent harness 的 *Rules-as-spec（1–7）+ Verification-loop（8）+ Memory（9）* 落到论文成稿。

## 第 0 层｜Phase 0：最小启动闸 + 完整 SPEC

Phase 0 只服务 A/B 完整成稿，并分两层：

- **最小启动闸**：主流程、问题、目标约束、结果状态。用于低成本启动、判断 A/B 是否可进入规划。
- **完整 SPEC**：正式逐节写正文前必须补齐。带 ⛔ 的项空着 = 禁止进入正式正文写作。

完整 SPEC 的关键项：

- ⛔ 目标约束（期刊：具体刊名；学位论文：学校规程 + 委员要求；日语：投稿刊/修論规程；图表：目标 house-style）
- ⛔ 档位现实性（期刊：贡献/数据/识别是否对得上；学位论文：章结构/材料/规程是否支撑提交）
- ⛔ 结构母本 / 规程母本 3–5 份
- ⛔ paper_type、method_family、primary estimand（不适用明确填 N/A）
- 一句话研究问题、一句话 contribution、本篇专属红线、主要/辅助识别、推断风险、外部有效性、PAP/预注册、数据访问与复现状态。

规则解释优先级：不造数/引用与主张上限 → 目标刊当前官方要求 → 目标刊动态层 → 论文类型/方法卡 → 通用默认值。

C–F（日语、图表、英文局部、投稿/R&R）属于局部/专项任务，使用轻量任务卡；只有任务扩大到整篇重构时才确认升级 A/B。

## 通用硬规则 1–9

| # | 规则 | 一句话 |
|---|---|---|
| 1 | 叙事框架 | 一篇 = 一个核心问题；按论文类型加载分节 playbook；每节开头一句"做什么、得什么"。 |
| 2 | 文献综述 = 定位 | 每簇收束到一句 gap；产出「文献地图」表，先有表再写散文；禁逐篇罗列。 |
| 3 | 新规性 / 贡献 | 1–3 条显式 bullet；每条对应真实检索；⛔ 禁裸 "we are the first to"；扛得住点名最近邻。 |
| 4 | **主张强度纪律** ★ | **结论语气 ≤ 识别强度**。关联/描述性设计全文禁因果措辞；按 method_family 加载识别策略卡。 |
| 5 | 引用诚信 | ⛔ 绝不编造作者/年份/标题/DOI；拿不准标【待核】；双时点核验（地基引用早查 + 投稿前穷尽）。 |
| 6 | 数据与可复现 / 图表先行 | 现代推断与复现审计 + 先设计整套图表再写散文；视觉自检；**渲染≠造数**。 |
| 7 | 文体 / 语域 | 简洁、主动语态、发现用现在时；hedging 纪律；⛔ 内部暗语禁入正文。 |
| 8 | 验证闸 | 《单节自查》每节跑；《总闸》投稿前跑；再跑引用穷尽核验 + 审稿人预演。 |
| 9 | 防漂移 / 记忆 | 每篇一份「成稿决策日志」；顶部钉死一句话问题；重大转向留痕。 |

## 三道验证闸

1. **《单节自查》**（每写完一节）——强度≤识别、引用真实、无内部暗语、图表过视觉自检。
2. **《总闸》**（投稿前）——研究问题没漂、贡献扛得住、无残留【待核】、无 over-claim、格式合规。
3. **《审稿人预演》**——Methodologist / Field Expert / Writing Critic 三视角，产出 Summary / Strengths / Blocker-Major-Minor / Missing evidence / 修改计划，不给虚假总分。

## 适用档位

期刊投稿档强调目标刊匹配、贡献现实性和投稿前终检；学位论文档强调学校规程、章结构完整性、材料充分性和答辩委员视角。两者都保留通用骨架（规则 1·2·4·5·7·8·9）。
