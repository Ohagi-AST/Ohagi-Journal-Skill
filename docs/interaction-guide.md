# 交互说明（面向人类用户）

完整成稿轨刻意做成**一次只推进一步**；日语、图表、英文局部和投稿任务使用专项任务卡。这里解释两种节奏如何切换。

## 核心节奏：问 → 等 → 执行

A/B 完整流程每一步，agent 会：

1. 先**渲染一次进度面板**（入口、档位、现在在哪、下一步是什么）；
2. **说清这步要做什么、需要你提供什么**；
3. **提问，然后停下等你回答/确认**；
4. 你确认后才执行，做完**再停**，把球交回给你。

这不是卡顿，是**防漂移**设计：长篇写作最大的敌人是中途目标悄悄偏移。每个 checkpoint 都拿正文跟「一句话研究问题」对一次。

C–F 局部/专项任务不展示完整 0A–6 面板。agent 只确认路径、输入事实、目标约束和允许范围，然后执行对应模块；一旦需要新增证据或重构整篇，必须先问你是否升级 A/B。

## 怎么低成本启动

要走 A/B 完整成稿时，先给四件事，agent 就能帮你判断下一步：

| 信息 | 例 |
|---|---|
| 主流程 | 英文期刊成稿 / 学位论文正文 |
| 一句话研究问题 | “本文研究 X 是否与 Y 相关，以及通过什么机制。” |
| 目标约束 | 目标刊、学校规程、日语投稿刊，或图表格式要求 |
| 结果状态 | 已有回归表、图表清单、日文草稿，或还缺什么 |

低成本启动只允许进入规划、路由、骨架讨论。**A/B 正式逐节写正文前，仍要补齐完整 SPEC**；C–F 只要求专项任务卡。

## 哪些地方它一定会停下等你（硬闸）

- **最小启动闸**：入口、问题、目标约束、结果状态不清楚时，不进入流程。
- **完整 SPEC 闸**：结构母本、contribution、识别红线、paper_type、method_family、primary estimand、推断/复现状态等没补齐时，不进入 A/B 正式正文逐节写作。
- **骨架确认**：章节骨架给你过目后才逐节写。
- **图表蓝图确认**：整套图表设计定稿后才逐节 write-to-exhibit。
- **journal-adapt 产物确认**：动态写作 skill 萃取出来要你过目。
- **每节定稿**：每写完一节停下，等你决定下一节。
- **总闸 / 引用核验 / 审稿人预演**：三道终检逐道停。

在这些点上，agent **不会替你跳闸**——你得明确说「确认 / 继续」。

## 你在每步大致要提供什么

| 步骤 | 你要给的 | 成本 |
|---|---|---|
| 最小启动闸 | 入口、问题、目标约束、结果状态 | 低 |
| 完整 SPEC | 目标刊/学校规程、结构母本、contribution、识别红线、引用样式、数据出处 | 中 |
| 文献宽度扩展（可选） | 你手头核心锚点论文（不到 10 篇，标题或 DOI） | 高 |
| 动态写作层（可选） | 目标刊 5–8 篇真论文 | 高 |
| 图表蓝图 | 你要用的结果/数据——哪些回归/图、各想说什么 | 中 |
| 逐节写 | 对草稿的反馈、确认进入下一节 | 中 |
| 日语优化 | 轨道：投期刊（哪本刊）还是修論（哪校规程） | 中 |
| 英文局部任务 | 原文/事实、目标刊、允许改动范围 | 低 |
| 投稿任务 | 目标刊/阶段、稿件或审稿意见、已完成的真实修改 | 中 |

## 怎么加速

- 想让它**连跑低风险步骤**：明确说「安装后的入口判断、最小启动闸、骨架初稿可以连着跑」。
- 想**跳过高成本可选项**：直接说「跳过文献宽度扩展 / 不需要 journal-adapt」。
- 想只做某个模块：直接说「只走图表蓝图」「只整理日语」「只改这一节英文」「只处理 R&R」。
- 不建议连跑：完整 SPEC 确认、图表蓝图确认、每节定稿、引用终检。这些是质量闸。

## 怎么叫停 / 转向

- 重大转向（换刊、换框架、砍章节）：直接说，agent 会**留痕**而不是静默改。
- 学位论文路径中，如果学校规程和期刊式建议冲突，**学校规程优先**。
- 日语整理中途发现结构本身不成立：agent 会提示「这超出日语整理范围，要不要转完整起草流程？」——**由你决定**，它不擅自升级。
- 图表蓝图入口中，如果发现主结果还没定，agent 会停在蓝图层，不会替你编结果。
- 英文局部任务中，如果缺系数/样本/引用，agent 会标 `【作者补】` / `【待核】`，不会生成看似合理的数字。
- R&R 中，如果作者没有提供新分析、页码或表号，agent 不会声称已经完成；任务扩大到全篇时由你确认是否升级。

## 进度面板看不懂？

A/B 使用完整《进度面板》；C–F 使用四步专项任务面板。模板见 [`SKILL.md`](../skills/journal-drafting/SKILL.md)。终端不支持 emoji 时完整流程会切到 ASCII 版（`[OK]/[..]/[ ]/[SK]`）。
