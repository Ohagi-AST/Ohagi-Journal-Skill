# 交互说明（面向人类用户）

这套 skill 刻意做成**一次只推进一步**。这页解释「为什么这么慢」「每步你要做什么」「怎么加速或叫停」。

## 核心节奏：问 → 等 → 执行

每一步，agent 会：

1. 先**渲染一次进度面板**（你现在在哪、下一步是什么）；
2. **说清这步要做什么、需要你提供什么**；
3. **提问，然后停下等你回答/确认**；
4. 你确认后才执行，做完**再停**，把球交回给你。

这不是卡顿，是**防漂移**设计：长篇写作最大的敌人是中途目标悄悄偏移。每个 checkpoint 都拿正文跟「一句话研究问题」对一次。

## 哪些地方它一定会停下等你（硬闸）

- **三个 ⛔**：目标期刊、目标层级现实性、结构母本 3–5 篇——没填满**不进入正文写作**。
- **骨架确认**：章节骨架给你过目后才逐节写。
- **图表蓝图确认**：整套图表设计定稿后才逐节 write-to-exhibit。
- **journal-adapt 产物确认**：动态写作 skill 萃取出来要你过目。
- **每节定稿**：每写完一节停下，等你决定下一节。
- **总闸 / 引用核验 / 审稿人预演**：三道终检逐道停。

在这些点上，agent **不会替你跳闸**——你得明确说「确认 / 继续」。

## 你在每步大致要提供什么

| 步骤 | 你要给的 |
|---|---|
| 第 0 层 SPEC | 目标刊名、对自己贡献/识别强度的诚实判断、3–5 篇结构母本 |
| 文献宽度扩展（可选） | 你手头核心锚点论文（不到 10 篇，标题或 DOI） |
| 图表蓝图 | 你要用的结果/数据——哪些回归/图、各想说什么 |
| 逐节写 | 对草稿的反馈、确认进入下一节 |
| 日语优化 | 轨道：投期刊（哪本刊）还是修論（哪校规程） |

## 怎么加速

- 想让它**连跑多步**：明确说「这几步连着跑，不用每步停」。否则它默认一步一停。
- 想**跳过某可选步骤**：直接说「跳过文献宽度扩展 / 不需要 journal-adapt」。

## 怎么叫停 / 转向

- 重大转向（换刊、换框架、砍章节）：直接说，agent 会**留痕**而不是静默改。
- 日语整理中途发现结构本身不成立：agent 会提示「这超出日语整理范围，要不要转完整起草流程？」——**由你决定**，它不擅自升级。
