# 快速开始（Getting Started）

最短路径：装好 skill → 选一个入口 → 把对应 prompt 改成你的情况 → 发给 agent。完整入口说明见 [`entrypoints.md`](entrypoints.md)。

## 1. 安装

见根 [`README.md`](../README.md) 的「怎么用 → 安装三个 skill」。装完 `journal-drafting`（原创编排器）即可起草；`vendor/` 下的 `reference-checker` 与 `journal-adapt` 在流程中按需被调用。

## 2. 六条快捷路径的可用 prompt

### A. 英文期刊成稿

```
我有一篇实证经济学论文要写成英文期刊稿。研究已经完成，有 [数据/回归结果/图表清单]。
目标期刊候选是 [Journal of ...]。
请走 journal-drafting 的英文期刊成稿入口，先帮我过最小启动闸，不要直接写正文。
```

适合：已有研究结果，想写成英文期刊论文。  
先准备：一句话研究问题、目标刊候选、3–5 篇结构母本、主要结果。

### B. 学位论文正文

```
我要写学位论文正文。研究已经完成，学校/研究科规程是 [规程或格式要求]。
我的研究问题是 [一句话问题]，已有结果包括 [主要结果/图表]。
请走 journal-drafting 的学位论文入口，先帮我搭章结构和最小启动闸。
```

适合：修士/博士论文或毕业论文。  
先准备：学校规程、导师/委员要求、章结构想法、已有结果。

### C. 已有日文稿整理

```
帮我整理这段日语。目标是 [投稿刊/修論规程]。
请走日语优化快速入口，只改语言表层和学术文体，不动论证、数据和引用内容：
[贴入日文段落]
```

适合：已有日文稿，只想去中文味、统一投稿轨/修論轨文体。  
Windows 下做字形检查时，优先把文本保存成 UTF-8 文件后运行 `python skills/journal-drafting/scripts/zh_glyph_check.py draft.txt`；不同终端的 `echo ... | python ...` 管道编码不一定稳定。

### D. 图表蓝图/图表规范

```
我想先做图表蓝图。这里是我的结果清单：[回归表/图/机制/稳健性清单]。
目标是 [期刊/学校规程]。
请只走 exhibit plan，帮我决定头牌图表、主文/附录放置、叙事弧和 write-to-exhibit 顺序。
```

适合：结果很多、图表顺序乱、还没决定哪些进正文/附录。  
先准备：每张表/图想承载的唯一论点、目标刊或学校格式。

### E. 英文局部任务

```
请走 journal-drafting 的英文局部快速入口，改写/审查下面这段 [abstract/introduction/results/conclusion]。
目标刊是 [Journal of ...]，论文类型/方法是 [...]。
只在我给出的事实内改写；缺数字写【作者补】，缺引用写【待核】，不要进入完整 SPEC：
[贴入文本或事实]
```

适合：只处理一段或一节，不想启动整篇成稿流程。

先准备：原文/真实结果、目标刊、允许改动范围。

### F. 投稿任务 / R&R

```
请走 journal-drafting 的投稿快速入口，处理下面的 [深度审查 / working-paper 缩稿 / R&R]。
目标期刊和阶段是 [...]。作者已经真实完成的修改是 [...]。
没有提供的新分析、数字、页码和表号一律标【作者补】：
[贴入稿件、编辑信或审稿意见]
```

适合：投稿前审查、缩稿或逐点回应审稿意见。

先准备：目标刊、当前稿件/意见、已完成的真实修改；精确投稿要求由 agent 查官方页面并记录核验日期。

## 3. 启动信息

### 完整成稿 A/B

材料不完整也可以先启动规划，但不能直接生成正式正文。先给 agent 这四件事：

1. 主流程：期刊稿或学位论文。
2. 一句话研究问题。
3. 目标刊、学校规程或日语轨道。
4. 研究结果是否已经基本完成。

A/B 正式逐节写作前仍要补齐完整 SPEC（结构母本、contribution、paper_type、method_family、primary estimand、识别红线、推断与复现状态等）。

### 局部/专项任务 C–F

只给任务路径、输入材料、目标约束和允许改动范围。skill 直接执行相应模块，不要求完整 SPEC；需要整篇重构、增加实质主张或补做分析时，才会确认是否升级 A/B。

## 4. 工作流卡片

| 阶段 | 用户输入 | 产出 | 成本 | 是否可跳过 |
|---|---|---|---|---|
| 最小启动闸 | A/B、问题、目标、结果状态 | 可启动判断 | 低 | A/B 否 |
| 完整 SPEC | 母本、贡献、paper_type、method_family、estimand、红线、推断/复现 | 本篇规则卡 | 中 | A/B 正式正文前否 |
| 文献宽度扩展 | 锚点论文 | 候选文献清单 | 高 | 可 |
| 动态写作层 | 目标刊 5–8 篇论文 | 动态写作 skill | 高 | 可 |
| 骨架 | 母本/规程/材料 | 章节空骨架 | 中 | 不建议 |
| 图表蓝图 | 结果清单 | exhibit 叙事弧 | 中 | 纯理论可改为命题蓝图 |
| 逐节写 | 反馈与确认 | 正文草稿 | 中 | 否 |
| 英文局部任务 | 原文/事实 + 允许范围 | 改写/审查 + Warnings | 低 | 独立入口 |
| 投稿任务 | 稿件/意见 + 真实修改 | 深审/缩稿/R&R | 中 | 独立入口 |
| 终检 | 齐稿与参考文献 | 修改优先级 | 中 | 学位论文可轻量 |
