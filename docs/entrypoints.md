# 入口选择（Entry Points）

先选入口，再进流程。不要一上来就把整套规则全跑完；这套 skill 会按你的入口自动跳过不适用的步骤。

| 入口 | 适合谁 | 先准备什么 | 第一条 prompt | 会走哪些阶段 | 可跳过 |
|---|---|---|---|---|---|
| A. 英文期刊成稿 | 实证经济学研究已完成，想写成英文期刊论文 | 一句话问题、目标刊候选、3–5 篇结构母本、主要结果/图表 | `我有一篇实证经济学论文要写成英文期刊稿。目标刊是 [...]。请走 journal-drafting 的英文期刊成稿入口，先帮我过最小启动闸。` | 最小启动闸 → 完整 SPEC → 动态写作层/文献宽度可选 → 骨架 → 图表蓝图 → 逐节写 → 三道终检 | 文献宽度扩展、journal-adapt 可跳过 |
| B. 学位论文正文 | 修士/博士论文或毕业论文，需要把研究写成正文 | 学校规程、导师/委员要求、章结构想法、已有结果 | `我要写学位论文正文。学校规程是 [...]，研究已经完成。请走 journal-drafting 的学位论文入口，先帮我搭章结构。` | 最小启动闸 → 学校规程版 SPEC → 章结构 → 图表蓝图 → 逐章/逐节写 → 总闸 → 答辩委员视角预演 | 目标刊贡献档位、journal-adapt、投稿体例 |
| C. 已有日文稿整理 | 已有日文草稿，只想去中文味/修学术文体 | 日文文本、投稿刊或学校规程 | `帮我整理这段日语。目标是 [投稿刊/修論规程]，请走日语优化快速入口，只改语言和文体，不动论证。` | 日语轨确认 → 字形/词汇/表达净化 → 投稿轨/修論轨文体优化 → 逐节确认 | Phase 0、骨架、逐节起草、终检 |
| D. 图表蓝图/图表规范 | 论文已大致有结构，想先整理结果图表和叙事 | 结果清单、回归/图、每张想说明什么、目标刊/学校格式 | `我想先做图表蓝图。这里是我的结果清单 [...]。请只走 exhibit plan，帮我决定主文/附录、头牌图表和 write-to-exhibit 顺序。` | 图表蓝图 → house-style → 视觉自检清单 → 可选渲染建议 | Phase 0 全流程、逐节正文、引用终检 |

## 一页式工作流卡片

| 阶段 | 目的 | 用户输入 | 产出 | 成本 | 可跳过 |
|---|---|---|---|---|---|
| 最小启动闸 | 判断能否进入规划 | 一句话问题、档位/入口、目标刊或学校规程、已有结果状态 | 可启动/需补材料的判断 | 低 | 不可跳过 |
| 完整 SPEC | 锁定正式写作边界 | 结构母本、贡献句、识别红线、引用样式、数据出处 | 本篇规则卡 | 中 | 正式正文前不可跳过 |
| 文献宽度扩展 | 补先行研究宽度 | 核心锚点论文 | 按簇候选文献清单 | 高 | 可跳过 |
| 动态写作层 | 学目标刊写作文化 | 目标刊 5–8 篇真论文 | `dynamic_writing_skill.md` | 高 | 可跳过 |
| 骨架 | 定章节顺序 | 母本/规程/研究材料 | 每节做什么、得什么 | 中 | 不建议跳过 |
| 图表蓝图 | 让图表先承载论点 | 结果/数据/图表清单 | exhibit 清单、叙事弧、主附放置 | 中 | 纯理论可改为命题蓝图 |
| 逐节写 | 生成正文 | 对每节草稿的反馈 | 正文草稿 + 自查 + warnings | 中 | 不可跳过 |
| 日语优化 | 去中文味、定文体 | 日文稿 + 投稿/修論轨 | 优化前后对照 | 中 | 仅日语轨 |
| 总闸/核验/预演 | 投稿或提交前查错 | 齐稿文本和参考文献 | 修改优先级清单 | 中 | 学位论文可轻量化 |

## 低成本启动

如果材料还不完整，也可以先走最小启动闸。只要能说清这四件事，就可以开始规划，但不能直接生成正式正文：

1. 这篇最终是期刊稿、学位论文、日语整理，还是图表蓝图？
2. 一句话研究问题是什么？
3. 目标刊/学校规程/日语轨道是什么？
4. 研究结果是否已经基本完成？

正式逐节写作前，仍必须补齐完整 SPEC；这是质量底线，不是流程负担。
