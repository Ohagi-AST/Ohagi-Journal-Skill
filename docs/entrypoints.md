# 入口选择（Entry Points）

先判断任务层级，再选快捷路径。用户意图明确时直接路由，不会为了分类多问一次。

## 第一层：完整成稿主流程

这两条路径交付整篇论文，使用最小启动闸、完整 SPEC、骨架、逐节写作与终检。

| 路径 | 适合谁 | 先准备什么 | 第一条 prompt | 会走哪些阶段 | 可跳过 |
|---|---|---|---|---|---|
| A. 英文期刊成稿 | 实证经济学研究已完成，想写成英文期刊论文 | 一句话问题、目标刊候选、3–5 篇结构母本、主要结果/图表 | `我有一篇实证经济学论文要写成英文期刊稿。目标刊是 [...]。请走 journal-drafting 的英文期刊成稿入口，先帮我过最小启动闸。` | 最小启动闸 → 完整 SPEC → 动态写作层/文献宽度可选 → 骨架 → 图表蓝图 → 逐节写 → 三道终检 | 文献宽度扩展、journal-adapt 可跳过 |
| B. 学位论文正文 | 修士/博士论文或毕业论文，需要把研究写成正文 | 学校规程、导师/委员要求、章结构想法、已有结果 | `我要写学位论文正文。学校规程是 [...]，研究已经完成。请走 journal-drafting 的学位论文入口，先帮我搭章结构。` | 最小启动闸 → 学校规程版 SPEC → 章结构 → 图表蓝图 → 逐章/逐节写 → 总闸 → 答辩委员视角预演 | 目标刊贡献档位、journal-adapt、投稿体例 |

## 第二层：局部/专项任务

这些是可独立调用的流程模块，使用轻量任务卡，不要求完整 SPEC。图表蓝图同时也是 A/B 结果型论文内部的标准阶段。

| 快捷路径 | 适合谁 | 先准备什么 | 第一条 prompt | 专项流程 | 升级条件 |
|---|---|---|---|---|---|
| C. 已有日文稿整理 | 已有日文草稿，只想去中文味/修学术文体 | 日文文本、投稿刊或学校规程 | `帮我整理这段日语。目标是 [投稿刊/修論规程]，请走日语优化快速入口，只改语言和文体，不动论证。` | 日语轨确认 → 字形/词汇/表达净化 → 投稿轨/修論轨文体优化 → 逐节确认 | 需要重搭结构或改论证时 |
| D. 图表蓝图/图表规范 | 论文已大致有结构，想先整理结果图表和叙事 | 结果清单、回归/图、每张想说明什么、目标刊/学校格式 | `我想先做图表蓝图。这里是我的结果清单 [...]。请只走 exhibit plan，帮我决定主文/附录、头牌图表和 write-to-exhibit 顺序。` | 图表蓝图 → house-style → 视觉自检清单 → 可选渲染建议 | 需要继续写整篇正文时 |
| E. 英文局部任务 | 只想改一段/一节、写摘要或快速审查，不重搭整篇 | 原文或真实事实、目标刊、允许改动范围、论文类型/方法（已知则填） | `请走英文局部快速入口，改写下面这段 results；保留原意，不补任何数字或引用：[文本]` | 轻量任务卡 → 事实缺口 → 改写/审查 → 改动说明+Warnings | 需要全篇重构或新增主张时 |
| F. 投稿任务 | 要深审、缩 working paper，或准备 R&R/referee response | 目标刊/阶段、全文或审稿意见、作者已完成的真实修改 | `请走投稿快速入口，逐点整理这份 referee report；没有的新分析、页码和表号一律标【作者补】：[...]` | 投稿任务卡 → 官方要求核验 → 审查/缩稿/R&R → 一致性检查 | 需要重写整篇或补做分析时 |

## 一页式工作流卡片

| 阶段 | 目的 | 用户输入 | 产出 | 成本 | 可跳过 |
|---|---|---|---|---|---|
| 最小启动闸 | 判断能否进入 A/B 规划 | 一句话问题、主流程、目标刊或学校规程、已有结果状态 | 可启动/需补材料的判断 | 低 | A/B 不可跳过 |
| 完整 SPEC | 锁定正式写作边界 | 结构母本、贡献句、识别红线、paper_type、method_family、primary estimand、推断/复现状态 | 本篇规则卡 | 中 | A/B 正式正文前不可跳过 |
| 文献宽度扩展 | 补先行研究宽度 | 核心锚点论文 | 按簇候选文献清单 | 高 | 可跳过 |
| 动态写作层 | 学目标刊写作文化 | 目标刊 5–8 篇真论文 | `dynamic_writing_skill.md` | 高 | 可跳过 |
| 骨架 | 定章节顺序 | 母本/规程/研究材料 | 每节做什么、得什么 | 中 | 不建议跳过 |
| 图表蓝图 | 让图表先承载论点 | 结果/数据/图表清单 | exhibit 清单、叙事弧、主附放置 | 中 | 纯理论可改为命题蓝图 |
| 逐节写 | 生成正文 | 对每节草稿的反馈 | 正文草稿 + 自查 + warnings | 中 | 不可跳过 |
| 日语优化 | 去中文味、定文体 | 日文稿 + 投稿/修論轨 | 优化前后对照 | 中 | 仅日语轨 |
| 英文局部任务 | 小范围改写或审查 | 原文/事实、目标、允许范围 | 改写/审查 + 改动说明 + Warnings | 低 | 独立快速入口 |
| 投稿任务 | 深审、缩稿或 R&R | 官方要求、稿件/意见、真实修改 | Blocker/Major/Minor 或逐点回应 | 中 | 独立快速入口 |
| 总闸/核验/预演 | 投稿或提交前查错 | 齐稿文本和参考文献 | 修改优先级清单 | 中 | 学位论文可轻量化 |

## 低成本启动

若要完整成稿，材料不完整也可先走最小启动闸。只要能说清这四件事，就可以开始规划，但不能直接生成正式正文：

1. 完整成稿走期刊稿还是学位论文？
2. 一句话研究问题是什么？
3. 目标刊/学校规程/日语轨道是什么？
4. 研究结果是否已经基本完成？

A/B 正式逐节写作前仍必须补齐完整 SPEC。C–F 只填轻量任务卡；若任务扩大到整篇重构、要新增实质主张或补做分析，agent 会先征得确认，再升级 A/B。
